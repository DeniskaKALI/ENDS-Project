# ТЕХНИЧЕСКОЕ ЗАДАНИЕ на разработку мобильного приложения «Моя полка»

---

## 1. ОБЩИЕ СВЕДЕНИЯ

**1.1. Наименование проекта:** Мобильное приложение для организации личного гардероба «Моя полка»

**1.2. Заказчик:** ФГАОУ ВО "Северо-Кавказский Федеральный Университет"

**1.3. Разработчик:** Астраков Борис Александрович

**1.4. Основание для разработки:** 
- Учебный план по направлению подготовки "Разработка и сопровождение программного обеспечения"
- Курсовой проект по дисциплине "Программная инженерия"

**1.5. Назначение разработки:**
Приложение предназначено для систематизации личного гардероба, упрощения процесса подбора ежедневных образов и повышения осознанности при управлении одеждой.

**1.6. Область применения:**
Приложение используется владельцами Android-устройств для ведения цифрового каталога одежды и аксессуаров, создания комплектов одежды и управления персональными настройками.

## 3. НАЗНАЧЕНИЕ И ЦЕЛИ РАЗРАБОТКИ

**3.1. Назначение системы:**
Система представляет собой клиент-серверное приложение, состоящее из:
- Мобильного приложения для Android (клиентская часть)

- Базы данных PostgreSQL

**3.2. Цели разработки:**
- Создание цифрового инструмента для каталогизации предметов гардероба

- Обеспечение офлайн-доступа к данным с последующей синхронизацией
- Реализация безопасной аутентификации пользователей
- Демонстрация применения современных технологий мобильной и серверной разработки

**3.3. Задачи разработки:**
- Реализация регистрации и аутентификации пользователей
- Создание CRUD-операций для управления вещами и образами

- Обеспечение локального кэширования и фоновой синхронизации
- Реализация персонализации интерфейса (темы, язык)

## 4. ТРЕБОВАНИЯ К СИСТЕМЕ

### 4.1. Требования к архитектуре

**4.1.1. Архитектурный паттерн:**
Система должна быть построена на основе адаптированного паттерна **PCMEF**:
- **Presentation** (клиент: Activities/Compose, ViewModel)
- **Control** (сервер: Controllers)
- **Mediator** (клиент: Repositories, сервер: Services)
- **Entity** (сервер: JPA Entities, клиент: Room Entities)
- **Foundation** (клиент: Room DAO, сервер: JPA Repositories)

**4.1.2. Клиент-серверная архитектура:**
- Взаимодействие через REST API
- Формат обмена данными: JSON
- Протокол: HTTP/HTTPS
- Аутентификация: JWT Bearer Token

### 4.5. Требования к базе данных

**4.5.1. СУБД:** PostgreSQL 14+

**4.5.2. Таблицы:**

| Таблица | Поля | Индексы |
|---------|------|---------|
| users | id (UUID, PK), email (UNIQUE), password_hash, display_name, avatar_url, created_at, updated_at | email |
| user_settings | user_id (UUID, PK, FK), theme, language, notifications_enabled, updated_at | - |
| items | id (UUID, PK), user_id (UUID, FK), name, description, image_url, category (ENUM), season (ENUM), created_at, updated_at | user_id |
| outfits | id (UUID, PK), user_id (UUID, FK), name, description, season (ENUM), created_at, updated_at | user_id |
| outfit_slots | id (UUID, PK), outfit_id (UUID, FK), item_id (UUID, FK, NULL), slot_type (ENUM) | outfit_id, item_id |

**4.5.3. Ограничения целостности:**
- CASCADE DELETE: users → items, outfits, user_settings
- SET NULL: items → outfit_slots.item_id
- CASCADE DELETE: outfits → outfit_slots
