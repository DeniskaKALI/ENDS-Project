# Спецификация основных прецедентов
| Параметр | UC-005: Управление вещами | UC-013: Управление образами | UC-020: Настроить приложение | UC-021: Управлять учётной записью |
|---|---|---|---|---|
| Актор | Пользователь | Пользователь | Пользователь | Пользователь |
| Описание | Добавление, просмотр, редактирование и удаление предметов гардероба с их метаданными. | Создание, просмотр, редактирование и удаление комплектов одежды (образов) из существующих вещей. | Изменение визуальных и функциональных параметров интерфейса приложения (тема, язык). | Изменение персональных данных профиля (аватар, имя, пароль) или удаление аккаунта. |
| Предусловия | Пользователь успешно аутентифицирован в системе. | 1. Пользователь аутентифицирован; 2. В каталоге есть хотя бы одна вещь (для создания образа). | Пользователь находится на вкладке «Настройки». | Пользователь находится в разделе «Управление аккаунтом» внутри настроек. |
| Постусловия | Изменения сохранены в локальной БД (Room) и отправлены в очередь на синхронизацию с сервером. | Образ сохранен в локальной БД и отправлен на синхронизацию. | Настройки применены немедленно и сохранены локально (тема) или на сервере (язык). | Данные профиля обновлены на сервере или аккаунт помечен на удаление. |
| Основной поток | 1. Пользователь открывает вкладку «Мои вещи»; 2. Система отображает список вещей; 3. Пользователь выбирает действие (добавить, редактировать, удалить); 4. Система сохраняет изменения и обновляет список. | 1. Пользователь открывает вкладку «Мои образы»; 2. Пользователь нажимает «Создать образ»; 3. Система открывает конструктор со слотами; 4. Пользователь заполняет слоты вещами из каталога; 5. Пользователь сохраняет образ. | 1. Пользователь выбирает опцию настройки (например, «Тема»); 2. Пользователь выбирает значение (Светлая/Тёмная); 3. Система мгновенно обновляет интерфейс приложения. | 1. Пользователь выбирает поле для редактирования (имя, пароль, аватар); 2. Вводит новые данные; 3. Система отправляет запрос на обновление профиля. |
| Альтернативные потоки | А1: Поиск и фильтрация. Пользователь вводит запрос или выбирает фильтр (Сезон/Категория). Система обновляет список согласно условиям. | А1: Редактирование образа. Пользователь выбирает существующий образ, меняет состав слотов или название, сохраняет изменения. | А2: Язык. Пользователь меняет язык интерфейса. Система перезагружает ресурсы и применяет новый язык. | А1: Смена пароля. Система запрашивает текущий пароль для подтверждения перед установкой нового. |
| Исключения | Е1: Ошибка сохранения. Если локальное хранилище переполнено, система показывает ошибку и предлагает очистить кэш. | Е1: Невозможно сохранить. Если образ пуст (нет вещей в слотах), система блокирует кнопку сохранения и показывает предупреждение. | Е1: Ошибка темы. Если применение темы вызывает сбой UI, система откатывается к системной теме по умолчанию. | Е1: Удаление аккаунта. При удалении аккаунта система требует повторного ввода пароля и предупреждает о потере всех данных. |
