# РАСШИРЕННЫЙ ГЛОССАРИЙ ПРОЕКТА

| Термин | Определение |
|--------|-------------|
| **Пользователь** | Физическое лицо, зарегистрированное в системе и использующее приложение для управления своим личным гардеробом |
| **Учётная запись** | Персональный профиль пользователя в системе, содержащий регистрационные данные (email, пароль, имя, аватар) и всю привязанную информацию о гардеробе |
| **Аутентификация** | Процесс проверки подлинности пользователя при входе в систему с помощью адреса электронной почты и пароля |
| **Регистрация** | Процесс создания новой учётной записи в системе с указанием обязательных данных (email, пароль, отображаемое имя) |
| **JWT-токен** | Цифровой токен безопасности (JSON Web Token), выдаваемый после успешной аутентификации для подтверждения личности пользователя при последующих запросах к серверу |
| **Вещь (Предмет гардероба)** | Отдельный элемент одежды или аксессуар, оцифрованный в системе с указанием фотографии, названия, описания, категории и сезонности |
| **Образ (Комплект одежды)** | Визуально сформированный набор вещей, собранных вместе в конструкторе для создания законченного стиля или наряда |
| **Категория вещи** | Классификационный признак предмета гардероба, определяющий его тип и место в структуре образа (головной убор, верх, низ, обувь, верхняя одежда, аксессуары) |
| **Сезонность** | Атрибут вещи или образа, указывающий на рекомендуемый временной период использования (зима, весна, лето, осень, внесезонное) |
| **Слот** | Структурированный блок в конструкторе образов, предназначенный для размещения вещи строго определённой категории |
| **Конструктор образов** | Визуальный инструмент приложения для создания комплектов одежды путём распределения вещей по предопределённым слотам |
| **Каталог вещей** | Раздел приложения, содержащий все добавленные пользователем предметы гардероба с функциями поиска и фильтрации |
| **Коллекция образов** | Раздел приложения, отображающий все созданные пользователем комплекты одежды в виде списка или сетки |
| **Поиск** | Функция нахождения вещей в каталоге по совпадению с введённым текстовым запросом (по названию или описанию) |
| **Фильтрация** | Функция отбора вещей или образов по заданным критериям (категория, сезонность, цвет) |
| **Локальное хранилище** | Встроенная база данных на устройстве пользователя (Room), сохраняющая вещи, образы и настройки для обеспечения работы приложения без подключения к интернету |
| **Офлайн-режим** | Функциональность приложения, позволяющая выполнять все основные операции с данными без активного подключения к интернету за счёт локального кэширования |
| **Синхронизация** | Фоновый процесс автоматического согласования данных между локальным хранилищем устройства и сервером при восстановлении интернет-соединения |
| **Сервер** | Удалённая серверная часть приложения на базе Java 17+ и Spring Boot, отвечающая за централизованное хранение данных, аутентификацию и бизнес-логику |
| **REST API** | Программный интерфейс взаимодействия между мобильным приложением и сервером, использующий протокол HTTP и формат обмена данными JSON |
| **OpenAPI / Swagger** | Стандарт описания и документирования REST API, позволяющий автоматически генерировать документацию по доступным эндпоинтам и форматам запросов |
| **PCMEF (Presentation-Controller-Mediator-Entity-Facade)** | Архитектурный паттерн, обеспечивающий строгое разделение ответственности между слоями: представление, управление состоянием, сетевой клиент, кэш, контроллеры, посредники, сущности и репозитории |
| **Retrofit** | Библиотека для Kotlin/Java, упрощающая выполнение сетевых запросов к REST API с поддержкой асинхронности и автоматической сериализации данных |
| **Room** | Библиотека абстракции над SQLite для Android, предоставляющая удобный API для работы с локальной базой данных и обеспечения типобезопасности запросов |
| **Spring Boot** | Фреймворк для Java, упрощающий создание серверных приложений с минимальной конфигурацией и встроенной поддержкой REST, безопасности и работы с базами данных |
| **Spring Data JPA** | Модуль Spring Boot для упрощения работы с базой данных через объектно-реляционное отображение (ORM) и репозитории |
| **Spring Security** | Фреймворк для реализации механизмов аутентификации и авторизации в приложениях на Spring, поддерживающий JWT и другие протоколы безопасности |
| **PostgreSQL** | Реляционная система управления базами данных с открытым исходным кодом, используемая на сервере для надёжного хранения пользовательских данных |
| **Тема оформления** | Визуальный стиль интерфейса приложения, определяющий цветовую схему элементов (светлая или тёмная) для комфортного использования в разных условиях освещения |
| **Аватар** | Персональное изображение профиля пользователя, отображаемое в учётной записи и используемое для визуальной идентификации |
| **Метаданные вещи** | Дополнительная структурированная информация о предмете гардероба: название, описание, категория, сезонность, дата добавления |
| **Навигация** | Система переключения между основными разделами приложения («Мои вещи», «Мои образы», «Настройки») через нижнюю панель вкладок |
| **Персонализация** | Настройка приложения под индивидуальные предпочтения пользователя: выбор темы, языка интерфейса, параметров уведомлений |
| **Клиент-серверная архитектура** | Модель взаимодействия, при которой мобильное приложение (клиент) отправляет запросы на удалённый сервер для выполнения операций с данными и получения ответов |
| **Агрегат (Aggregate)** | Группа связанных объектов, рассматриваемых как единое целое для целей изменения данных; в проекте агрегатом является пользователь, владеющий вещами и образами |
| **Сущность (Entity)** | Объект предметной области, имеющий уникальный идентификатор и жизненный цикл, не зависящий от атрибутов (например, пользователь, вещь, образ) |
| **Репозиторий (Repository)** | Слой абстракции для доступа к данным, инкапсулирующий логику работы с базой данных (локальной или серверной) и предоставляющий методы для операций CRUD |
| **Контроллер (Controller)** | Компонент серверной части, обрабатывающий входящие HTTP-запросы, вызывающий бизнес-логику и возвращающий ответы в формате JSON |
| **Посредник (Mediator)** | Компонент архитектуры PCMEF, координирующий взаимодействие между различными модулями системы и управляющий потоками данных |
| **Фасад (Facade)** | Компонент, предоставляющий упрощённый интерфейс к сложной подсистеме, скрывающий детали реализации от вышележащих слоёв |
| **End-to-End тестирование** | Тип тестирования, проверяющий работу всего приложения от пользовательского интерфейса до серверной части в условиях, приближённых к реальным |
| **Покрытие кода тестами** | Метрика, показывающая процент строк кода, выполняемых при запуске автоматизированных тестов; для проекта установлено минимальное требование >40% |
