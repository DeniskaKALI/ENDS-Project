# ПАСПОРТ ПРОЕКТА

## Основная информация

| Параметр | Значение |
|----------|----------|
| **Название проекта** | Мобильное приложение для организации личного гардероба «Моя полка» |
| **Тип** | Курсовой проект с клиент-серверной архитектурой |
| **Цель проекта** | Создание цифрового инструмента для систематизации личного гардероба, упрощения процесса подбора ежедневных образов и повышения осознанности при управлении одеждой |

## Актуальность и проблема

Пользователи регулярно сталкиваются с трудностями при организации хранения вещей, забывают о наличии конкретных предметов в шкафу и тратят значительное время на утренний подбор комплектов. Отсутствие структурированного цифрового каталога приводит к нерациональному использованию гардероба и затрудняет планирование стиля. 

Проект устраняет этот разрыв за счёт переноса физического шкафа в интуитивный и доступный мобильный интерфейс с возможностью визуального конструирования образов и офлайн-доступа.

## Описание решения

Приложение позволяет:
- Оцифровывать предметы одежды, добавляя фотографии и метаданные
- Собирать готовые комплекты через визуальный конструктор с фиксированными слотами для разных категорий одежды
- Работать автономно благодаря локальному кэшированию и фоновой синхронизации с сервером при появлении подключения к интернету

**Навигация** построена вокруг трёх ключевых разделов:
1. Каталог вещей
2. Коллекция образов
3. Параметры приложения

Интерфейс адаптирован под светлую и тёмную темы оформления.

## Целевая аудитория

Пользователи Android-устройств любого возраста и пола, интересующиеся:
- Модой
- Систематизацией быта
- Минимализмом
- Планированием гардероба

## Ключевые функции

- Безопасная регистрация и вход  
- Создание карточек вещей с указанием категории, сезона и описания  
- Визуальный сбор образов через структурированные блоки  
- Просмотр детальной информации, редактирование и удаление элементов  
- Управление учётной записью и персонализация интерфейса  
- Работа без подключения к сети с последующей автоматической синхронизацией  

## Техническая реализация

### Клиентская часть (Android)
| Технология | Назначение |
|------------|------------|
| **Kotlin** | Основной язык разработки |
| **Android Studio** | Среда разработки |
| **Jetpack Compose / XML** | Построение пользовательского интерфейса |
| **Retrofit** | Выполнение сетевых запросов |
| **Room** | Локальное хранение данных |

### Серверная часть
| Технология | Назначение |
|------------|------------|
| **Java 17+** | Язык программирования |
| **Spring Boot** | Фреймворк для создания backend-приложения |
| **PostgreSQL** | Реляционная база данных |
| **Spring Data JPA** | Работа с данными |
| **Spring Security + JWT** | Аутентификация и защита данных |

### Интеграция
- **REST API** — взаимодействие между клиентом и сервером
- **OpenAPI / Swagger** — документация API
- **Архитектура**: адаптированный паттерн **PCMEF** (Presentation, Control, Mediator, Entity, Framework) с разделением на слои:
  - Представления
  - Управления состоянием
  - Сетевого клиента
  - Кэша
  - Контроллеров
  - Посредников
  - Сущностей
  - Репозиториев
