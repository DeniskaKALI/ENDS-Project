# БИЗНЕС-ГЛОССАРИЙ ПРОЕКТА

| Термин | Определение |
|--------|-------------|
| **Пользователь** | Физическое лицо, зарегистрированное в системе и использующее приложение для управления своим гардеробом |
| **Учётная запись** | Персональный профиль пользователя в системе, содержащий регистрационные данные (email, пароль, имя, аватар) и всю информацию о гардеробе |
| **Вещь (предмет гардероба)** | Отдельный элемент одежды или аксессуар, оцифрованный в системе с указанием фотографии, названия, описания, категории и сезонности |
| **Образ (комплект одежды)** | Визуально сформированный набор вещей, собранных вместе для создания законченного стиля или наряда |
| **Категория вещи** | Классификационный признак предмета гардероба (головной убор, верхняя одежда, обувь, аксессуары и т.д.) |
| **Сезонность** | Атрибут вещи или образа, указывающий на временной период использования (зима, весна, лето, осень, внесезонное) |
| **Слот** | Структурированный блок в конструкторе образов, предназначенный для размещения вещи определённой категории |
| **Конструктор образов** | Визуальный инструмент приложения для создания комплектов одежды путём распределения вещей по слотам |
| **Каталог вещей** | Раздел приложения, содержащий все добавленные пользователем предметы гардероба с возможностью поиска и фильтрации |
| **Коллекция образов** | Раздел приложения, отображающий все созданные пользователем комплекты одежды |
| **Аутентификация** | Процесс проверки подлинности пользователя при входе в систему с помощью email и пароля |
| **Регистрация** | Процесс создания новой учётной записи в системе с указанием email, пароля и отображаемого имени |
| **JWT-токен** | Цифровой токен безопасности, выдаваемый после успешной аутентификации для подтверждения личности пользователя при запросах к серверу |
| **Синхронизация** | Процесс автоматического согласования данных между локальным хранилищем устройства и сервером при наличии интернет-соединения |
| **Офлайн-режим** | Функциональность приложения, позволяющая работать с данными без подключения к интернету за счёт локального кэширования |
| **Локальное хранилище** | База данных на устройстве пользователя (Room), сохраняющая вещи, образы и настройки для работы без интернета |
| **Сервер** | Удалённая серверная часть приложения на Java/Spring Boot, отвечающая за хранение данных, аутентификацию и бизнес-логику |
| **REST API** | Программный интерфейс взаимодействия между мобильным приложением и сервером, использующий HTTP-протокол и формат JSON |
| **Тема оформления** | Визуальный стиль интерфейса приложения (светлая или тёмная цветовая схема) |
| **Аватар** | Персональное изображение профиля пользователя в учётной записи |
| **Метаданные вещи** | Дополнительная информация о предмете гардероба (название, описание, категория, сезонность, цвет, размер) |
| **Навигация** | Система переключения между основными разделами приложения («Мои вещи», «Мои образы», «Настройки») через нижнюю панель |
| **Поиск** | Функция нахождения вещей в каталоге по названию или другим атрибутам |
| **Фильтрация** | Функция отбора вещей или образов по заданным критериям (категория, сезонность и др.) |
| **Персонализация** | Настройка приложения под предпочтения пользователя (тема, язык, уведомления, параметры учётной записи) |
