## 📋 Описание проекта

**Progile** — это клиент-серверная система для мониторинга транспорта, управления маршрутами и анализа логистических процессов.

Проект предназначен для организаций, которым необходимо контролировать транспортные средства, отслеживать выполнение маршрутов, фиксировать GPS-данные, анализировать рейсы и формировать отчеты.

Приложение позволяет:

* 🚚 просматривать список транспортных средств;
* 🗺️ управлять маршрутами и назначать транспорт на рейсы;
* 📍 фиксировать GPS-снимки и отслеживать местоположение транспорта;
* 📊 формировать отчеты по маршрутам и поездкам;
* 💬 добавлять комментарии к транспортным средствам и маршрутам;
* 👥 работать с пользователями и ролями;
* 🔐 безопасно выполнять вход через JWT-аутентификацию;
* 📡 получать данные через REST API;
* 📱 использовать мобильный интерфейс для диспетчеров, логистов, водителей, администраторов и руководителей.

Целевая аудитория проекта — диспетчеры, логисты, водители, администраторы и руководители транспортных или логистических организаций.

## 🏗️ Архитектура

Система построена на архитектурном паттерне **PCMEF**:

**PCMEF** — это архитектурный подход, разделяющий систему на пять слоев:

| Слой                 | Технология                    | Ответственность                                       |
| -------------------- | ----------------------------- | ----------------------------------------------------- |
| **Presentation (P)** | Android / Jetpack Compose     | Пользовательский интерфейс, экраны, ввод данных       |
| **Control (C)**      | Spring Boot REST Controllers  | Обработка HTTP-запросов, REST API, валидация DTO      |
| **Mediator (M)**     | Spring Boot Services          | Бизнес-логика, координация операций, транзакции       |
| **Entity (E)**       | JPA Entities / DTO            | Сущности предметной области и объекты передачи данных |
| **Foundation (F)**   | Spring Data JPA, Repositories | Доступ к базе данных и работа с PostgreSQL            |

Такое разделение позволяет упростить сопровождение проекта, снизить связанность компонентов и сделать архитектуру более понятной и расширяемой.

Подробнее в документации: [Архитектурное проектирование](docs/02-architecture/)

## 🗄️ База данных

В проекте используется **PostgreSQL**.

Основные таблицы системы:

* `user_accounts` — учетные записи пользователей;
* `transport` — транспортные средства;
* `routes` — маршруты;
* `tracking_snapshots` — GPS-снимки и телеметрия;
* `reports` — отчеты по рейсам;
* `comments` — комментарии пользователей;
* `monitored_entities` — универсальная сущность для обязательного CRUD API.

Миграции базы данных выполняются автоматически при запуске backend-приложения с помощью Flyway.

## 🔐 Безопасность

В проекте реализованы основные механизмы безопасности:

* **JWT-аутентификация** — выдача и проверка токенов доступа;
* **Spring Security** — защита REST API и разграничение доступа;
* **BCrypt** — хеширование паролей пользователей;
* **Role-based access control** — управление доступом на основе ролей;
* **DTO-валидация** — проверка входных данных;
* **Защита от SQL-инъекций** — использование Spring Data JPA и параметризованных запросов;
* **HTTPS** — защита передаваемых данных при production-развертывании.

Подробнее: [Архитектурные решения ADR](docs/02-architecture/ADR.md)

* Offline-режим может быть ограничен кэшированием последних загруженных данных.

* Для запуска на физическом устройстве требуется корректно указать IP-адрес backend-сервера.
