# Архитектурные решения Progile

## ADR-01. Kotlin и Jetpack Compose

Клиент реализован как нативное Android-приложение на Kotlin. Jetpack Compose и Material 3 обеспечивают декларативный UI, единую тему и адаптивную вёрстку. Проект открывается и собирается непосредственно в Android Studio.

## ADR-02. MVVM и StateFlow

`ProgileViewModel` хранит состояние авторизации, навигации, транспорта, маршрутов и настроек. Composable-функции получают неизменяемый `AppUiState` через `StateFlow`, что отделяет UI от бизнес-состояния.

## ADR-03. Локальная регистрация для прототипа

Учётные записи прототипа сохраняются в `SharedPreferences`. Пароль хранится как salted SHA-256 хеш. Демонстрационная учётная запись определяется по заранее вычисленному хешу, открытый пароль в коде не хранится. Для промышленной версии предусмотрена замена локального репозитория на JWT API backend.

## ADR-04. DemoTransportRepository

Демонстрационные транспорт и маршруты доступны только специальной учётной записи. Новые локальные пользователи получают пустой набор данных. Интерфейс `TransportRepository` позволяет позднее подключить REST API без переписывания экранов.

## ADR-05. OpenStreetMap без API-ключа

Реальная карта загружается через HTTPS в системном `WebView`. При отсутствии сети автоматически отображается локальная Canvas-схема, поэтому основной интерфейс остаётся работоспособным офлайн.

## ADR-06. Spring Boot, JWT и PostgreSQL

Сервер реализован отдельно на Java 17 и Spring Boot 3. Он предоставляет REST API, OpenAPI, JWT-аутентификацию, BCrypt и JPA-доступ к PostgreSQL. Такое разделение позволяет независимо демонстрировать клиентский прототип и серверный контракт.
