# Устав проекта Progile

## Резюме

Progile - мобильное приложение для мониторинга транспорта, управления маршрутами и формирования отчетов по перевозкам. Проект решает проблему ручного контроля рейсов, разрозненной коммуникации и отсутствия единой картины по состоянию транспортного парка.

## Цель

Создать мобильный клиент и серверную часть, которые позволяют регистрировать пользователей, разграничивать роли, вести транспорт, отслеживать статусы и GPS-снимки, назначать маршруты, добавлять комментарии, формировать отчеты и работать с кэшем без интернета.

## Границы первой версии

Входит:

- Kotlin/Jetpack Compose Android UI с 7 экранами и нижней навигацией;
- Java + Spring Boot backend;
- REST API, включая обязательные `/api/entities`;
- JWT, BCrypt и роли `USER`, `DISPATCHER`, `LOGIST`, `ADMIN`, `MANAGER`, `DRIVER`;
- PostgreSQL для учетных записей и универсальных сущностей;
- OpenAPI/Swagger UI;
- локальный DemoTransportRepository, SharedPreferences с salted хешами и Canvas-fallback карты;
- модульные тесты backend с JaCoCo-порогом 40%.

Не входит в первую версию: ИИ-прогнозирование маршрутов, VR/AR, государственные интеграции и промышленная телематика.

## Критерии успеха

- Android-пользователь может зарегистрироваться и войти локально, а серверная аутентификация возвращает JWT;
- API содержит больше 8 endpoint и доступно через Swagger;
- UI содержит больше 5 экранов и соответствует Material Design;
- демонстрационные транспорт и маршруты доступны без сети, а карта имеет локальный fallback;
- backend собирается в JAR и проходит тесты.
