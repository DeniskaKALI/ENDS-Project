[Normal] 5.1 Паттерн Strategy
[Normal] Описание: Паттерн Strategy используется для определения различных стратегий валидации переходов статусов транспортных средств в зависимости от типа парка (легковой, грузовой, спецтехника) или приоритета маршрута.
[Normal] // Интерфейс стратегии валидации статусов транспорта
[Normal] public interface TransportStatusValidationStrategy {
[Normal] boolean canTransition(TransportStatus current, TransportStatus next);
[Normal] List<TransportStatus> getAllowedNextStatuses(TransportStatus current);
[Normal] }
[Normal] // Стандартная стратегия (линейный workflow для обычных ТС)
[Normal] public class StandardTransportWorkflowStrategy implements TransportStatusValidationStrategy {
[Normal] private static final Map<TransportStatus, List<TransportStatus>> ALLOWED_TRANSITIONS = Map.of(
[Normal] TransportStatus.MOVING, List.of(TransportStatus.STOPPED, TransportStatus.OFF_ROUTE),
[Normal] TransportStatus.STOPPED, List.of(TransportStatus.MOVING, TransportStatus.MAINTENANCE),
[Normal] TransportStatus.OFF_ROUTE, List.of(TransportStatus.MOVING, TransportStatus.STOPPED),
[Normal] TransportStatus.MAINTENANCE, List.of(TransportStatus.STOPPED),
[Normal] TransportStatus.STOPPED, List.of(TransportStatus.MOVING, TransportStatus.MAINTENANCE)
[Normal] );
[Normal] @Override
[Normal] public boolean canTransition(TransportStatus current, TransportStatus next) {
[Normal] return ALLOWED_TRANSITIONS.getOrDefault(current, List.of()).contains(next);
[Normal] }
[Normal] }
[Normal] // Стратегия для спецтехники (с дополнительным статусом REPAIR)
[Normal] public class HeavyVehicleWorkflowStrategy implements TransportStatusValidationStrategy {
[Normal] private static final Map<TransportStatus, List<TransportStatus>> ALLOWED_TRANSITIONS = Map.of(
[Normal] TransportStatus.MOVING, List.of(TransportStatus.STOPPED, TransportStatus.OFF_ROUTE),
[Normal] TransportStatus.STOPPED, List.of(TransportStatus.MOVING, TransportStatus.MAINTENANCE, TransportStatus.REPAIR),
[Normal] TransportStatus.REPAIR, List.of(TransportStatus.MAINTENANCE, TransportStatus.STOPPED),
[Normal] TransportStatus.OFF_ROUTE, List.of(TransportStatus.MOVING, TransportStatus.STOPPED),
[Normal] TransportStatus.MAINTENANCE, List.of(TransportStatus.STOPPED, TransportStatus.MOVING)
[Normal] );
[Normal] @Override
[Normal] public boolean canTransition(TransportStatus current, TransportStatus next) {
[Normal] return ALLOWED_TRANSITIONS.getOrDefault(current, List.of()).contains(next);
[Normal] }
[Normal] }

[Обычный точно] Ответы на контрольные вопросы:
[List Paragraph] Что такое диаграмма последовательности и для чего она используется?
[Normal] Диаграмма последовательности - это диаграмма UML, показывающая взаимодействие объектов во времени. Она используется для детализированного проектирования поведения системы, показывая, какие объекты взаимодействуют, в каком порядке и какие сообщения обмениваются.
[List Paragraph] Назовите основные элементы диаграммы последовательности.
[Normal] Основные элементы: акторы (участники), линии жизни (представляют объекты), сообщения (стрелки, указывающие вызов методов), возврат (результаты вызова), фрагменты (условные и циклические блоки), создание и уничтожение объектов.
[List Paragraph] Как на диаграмме последовательности показать условное выполнение?
[Normal] Условное выполнение показывается через фрагмент alt (альтернатива), который разделяет выполнение на разные ветви в зависимости от условий.
[List Paragraph] Как показать цикл на диаграмме последовательности?
[Normal] Цикл показывается через фрагмент loop, который обвивает действия, повторяющиеся до выполнения определенного условия.
[List Paragraph] В чем разница между синхронным и асинхронным сообщением?
[Normal] Синхронное сообщение блокирует вызывающий объект до получения ответа, а асинхронное - нет, выполнение продолжается без ожидания ответа.
[List Paragraph] Какие методы выявляются при построении диаграмм последовательности?
[Normal] Выявляются методы, которые объекты вызывают друг у друга, параметры, которые передаются в методы, и возвращаемые значения.
[List Paragraph] Как диаграммы последовательности связаны с архитектурой PCMEF?
[Normal] Диаграммы последовательности в PCMEF показывают взаимодействие между слоями системы: Presentation, Control, Mediator, Entity, Foundation и Database, поддерживая архитектурные принципы.
[List Paragraph] Почему важно соблюдать направление вызовов сверху вниз (P - C - M - E - F)?
[Normal] Это важно для соблюдения четкой структуры архитектуры, где каждый слой взаимодействует с нижележащими слоями, что упрощает поддержку и расширяемость системы.
[List Paragraph] Как на диаграмме последовательности показать создание нового объекта?
[Normal] Создание нового объекта показывается специальным маркером (например, символом "new" или прямоугольником с надписью "new"), который обозначает, что объект был создан в процессе выполнения.
[List Paragraph] Какие паттерны GoF наиболее часто используются при детальном проектировании?
[Normal] Часто используются паттерны Strategy (для выбора алгоритмов), Observer (для уведомлений), Factory Method (для создания объектов), Command (для инкапсуляции запросов) и Singleton (для обеспечения единственного экземпляра).
[List Paragraph] Как обеспечить соответствие между диаграммами последовательности и диаграммой классов?
[Normal] Для этого нужно, чтобы методы, вызванные в диаграммах последовательности, были реализованы в классах диаграммы классов, а связи между объектами и их взаимодействия были соответствующим образом отражены.
[List Paragraph] Какие проблемы могут возникнуть, если не построить диаграммы последовательности до реализации?
[Normal] Без диаграмм последовательности могут возникнуть проблемы с неправильной синхронизацией взаимодействий объектов, трудности в выявлении и исправлении ошибок, а также нарушения архитектурных принципов.
[List Paragraph] Как проверить, что диаграмма последовательности не нарушает архитектурные принципы?
[Normal] Необходимо проверить, чтобы все вызовы и взаимодействия объектов следовали архитектурным слоям и направлениям вызова, соответствовали принципам инкапсуляции и разделения ответственности.
