[Heading 1] Заключение
[Normal] В ходе выполнения работы был разработан веб-интерфейс для программной системы на основе спроектированной архитектуры PCMEF, используя React для построения пользовательского интерфейса и реализованный ранее Java бэкенд (Control/Mediator/Entity/Foundation) через REST API
[Heading 1] Ответы на контрольные вопросы:
[List Paragraph] Взаимодействие React с Java-бэкендом организуется через HTTP-запросы. React отправляет запросы к API, реализованному на Spring Boot, получает данные в формате JSON и отображает их в интерфейсе.
[List Paragraph] REST API — это способ обмена данными между клиентом и сервером через HTTP-методы (GET, POST, PUT, DELETE). В проекте React обращается к REST API для получения, создания, изменения и удаления данных.
[List Paragraph] CORS позволяет браузеру выполнять запросы между разными доменами. В Spring Boot CORS настраивается через аннотацию @CrossOrigin или глобальную конфигурацию, где указывается адрес React-приложения.
[List Paragraph] Аутентификация с JSON Web Token реализуется так: пользователь отправляет логин и пароль, сервер возвращает JWT, React сохраняет токен в localStorage или sessionStorage и использует его для последующих запросов.
[List Paragraph] Axios interceptors позволяют автоматически добавлять JWT в заголовок Authorization перед каждым запросом, что избавляет от необходимости вручную передавать токен.
[List Paragraph] React Router отвечает за маршрутизацию в одностраничном приложении. С его помощью задаются URL-пути и соответствующие им компоненты.
[List Paragraph] Защита маршрутов реализуется через компонент PrivateRoute, который проверяет наличие JWT. Если пользователь не авторизован, происходит перенаправление на страницу входа.
[List Paragraph] React Hook Form упрощает работу с формами: хранит значения полей, выполняет валидацию и обрабатывает отправку данных без большого количества useState.
[List Paragraph] Асинхронные запросы выполняются внутри useEffect, а результат сохраняется через useState. После получения ответа React автоматически обновляет интерфейс.
[List Paragraph] Индикатор загрузки отображается с помощью состояния loading. Перед отправкой запроса оно устанавливается в true, после завершения — в false, и в зависимости от значения показывается спиннер или основной контент.
